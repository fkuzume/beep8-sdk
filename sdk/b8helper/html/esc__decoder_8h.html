Detailed Description

Module for decoding and handling escape sequences.

This module provides functionality for decoding escape sequences using the CEscapeSeqDecoder class and associated functions. Escape sequences are special strings used primarily for terminal display control, such as moving the cursor or clearing the screen.

Overview of Features

<tt>CEscapeSeqDecoder</tt> Class

  • Stream Method: Parses the input code (character) and determines the corresponding operation, storing it in the EscapeOut structure. This method uses a state machine to handle continuous escape sequences.
  • Constructor and Destructor: Initializes and cleans up the class.

<tt>EscapeOut</tt> Structure

This structure holds the result of parsing an escape sequence, including:

  • _code: The parsed code.
  • _EscapePAL: Information about the palette selection.
  • _Ope: The operation corresponding to the escape sequence.
  • _x, _y: Cursor positions.

Helper Functions

  • EscClearEntireScreen: Function to clear the entire screen.
  • EscMoveCursor: Function to move the cursor to a specified position.

Detailed Description of <tt>CEscapeSeqDecoder</tt> Class

The CEscapeSeqDecoder class contains an internal ImplCEscapeSeqDecoder class that manages the state and parsing of escape sequences. It uses a state machine to parse sequences and determine the appropriate operations. The states are defined in the EscState enumeration, and each state handles specific characters accordingly.

Usage Example

Here is an example of how to use this module:

#include "esc_decoder.h"
int main() {
CEscapeSeqDecoder decoder;
FILE* fp = stdout;
// Clear the entire screen
EscClearEntireScreen(fp);
// Move the cursor
EscMoveCursor(fp, 10, 5);
// Process escape sequence
const char* esc_seq = "\033[2J";
for (const char* p = esc_seq; *p; ++p) {
EscapeOut result = decoder.Stream(*p);
// Execute operations using result if needed
}
return 0;
}

Detailed Functionality

State Machine Implementation

The Stream method processes each character of the escape sequence and uses a state machine to determine the appropriate action based on the current state. For example, when the escape character (0x1b) is received, the state transitions to ES_0x1b, and the following characters are processed accordingly. If [ follows, the state transitions to ES_0x5b, and the sequence is further analyzed to determine the exact operation.

Execution of Operations

The operations are defined in the EscapeSeqOpe enumeration and include actions like moving the cursor (ESO_MOVE_CURSOR) or clearing the screen (ESO_CLEAR_ENTIRE_SCREEN). These operations are stored in the EscapeOut structure and can be executed as needed.

Summary

This module provides a decoder for escape sequences, enabling terminal display control and cursor operations through escape sequences. The state machine-based parsing allows handling complex escape sequences efficiently.
